# KiCad Studio 2.6.0 Release Notes

KiCad Studio 2.6.0 hardens the optional `kicad-mcp-pro` integration while keeping the core KiCad workspace features independent. Viewers, exports, DRC/ERC via `kicad-cli`, BOM/netlist, language services, and library search still work when MCP is not installed or not reachable.

## MCP Compatibility

The extension now negotiates the MCP server version during initialize. It supports `kicad-mcp-pro >=3.0.0 <4.0.0`, recommends `>=3.0.2 <4.0.0`, and records the last observed server card for optimistic UI rendering on the next launch. Incompatible servers are marked explicitly and do not receive context pushes, fix queue calls, quality gate calls, or release requests.

## Quality Gates Sidebar

The new Quality Gates view surfaces project, placement, transfer, and manufacturing readiness checks. It renders cached gate results while fresh MCP calls are running, expands failed or blocked rows into violations, and opens source locations when the server provides file and line metadata.

## Fix Queue Code Actions

MCP fix queue items with `path` and `line` metadata now appear as VS Code Code Actions near matching diagnostics. Text-only fix queue items continue to render in the AI Fix Queue tree and can still be applied manually.

## Installer And Profiles

`KiCad: Install kicad-mcp-pro` detects `uvx`, `pipx`, and `pip`, then runs the selected installer as a visible VS Code Task. The MCP status entry also exposes a profile picker for the documented `full`, `minimal`, `schematic_only`, `pcb_only`, `manufacturing`, `high_speed`, `power`, `simulation`, `analysis`, and `agent_full` profiles.

## Manufacturing Release Wizard

`KiCad: Manufacturing Release Wizard` guides users through variant selection, project quality pre-flight checks, output confirmation, MCP manufacturing export, and post-run handling. Blocking gates surface MCP hints when present.

## Configuration And Debugging

`.vscode/mcp.json` now has KiCad-aware JSON schema validation. The new MCP log viewer keeps a redacted ring buffer of recent request/response traffic and can save that log for bug reports without exposing authorization headers or full home-directory paths.

## CI Coverage

The release adds real-server integration test scaffolding that can launch `kicad-mcp-pro==3.0.2` through `uvx` on Linux CI when policy allows it. Mocked unit and integration tests continue to cover version negotiation, tree mapping, Code Actions, installer selection, structured errors, and log redaction.
